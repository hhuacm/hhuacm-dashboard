# HHUACM Dashboard 设计准则

HHUACM Dashboard 是面向竞赛与团队运营的后台产品。

目标气质：清爽、可靠、克制、精确、蓝白、低装饰。

核心体验：用户能快速判断状态、找到入口、完成操作。

参考成熟后台或工具类产品时（如 Vercel），只提取克制感、层级感、组件完成度和单主色策略，不复制它们的专有图形、行业元素或页面结构。

## 页面类型

### 公开入口

公开入口可以更有识别度，但必须服务登录、注册和产品定位。

- 可以使用较大的标题。
- 可以使用更明显的蓝白视觉。
- 主操作必须清楚。
- 表单、面板、按钮使用统一基础组件。

### 后台页面

后台页面必须优先呈现任务、状态和数据。

- 使用稳定的应用壳：顶部栏、侧边栏或二者组合。
- 页面标题说明当前位置和主要任务。
- 内容区优先使用状态卡片、表格、筛选、表单、详情面板。
- 空状态、加载状态、错误状态必须真实存在。
- 不使用巨幅首屏、宣传口号、假产品预览、无意义进度条。

## 组件策略

基础控件优先使用 HeroUI V3 React 组件，再按本项目的 HeroUI/Tailwind Design tokens 收敛颜色、圆角、尺寸和状态。

不堆叠一次性样式，而是复用 HeroUI V3 compound components、BEM classes、语义 token、键盘交互和状态处理。

扩展基础控件时，优先覆盖 HeroUI token 或 BEM class，不在页面中堆一次性 Tailwind class。

业务组件可以自定义，但必须组合基础控件。业务组件负责信息结构，基础组件负责交互一致性。

## 颜色

以蓝白作为主色调（是方向，不是把所有元素染蓝）。

### 色彩角色

| 角色 | 用途 |
| --- | --- |
| 页面背景 | 接近白的冷色浅蓝，承托内容 |
| 卡片表面 | 白色或极浅冷灰，承载信息 |
| 主文本 | 深蓝黑或近黑，表达标题和关键值 |
| 次文本 | 冷灰蓝，表达描述和元信息 |
| 主色 | 天蓝或钻石蓝，用于主操作和关键强调 |
| 主色深色 | 悬停、按下、强调文字 |
| 主色浅色 | 选中背景和轻量提示 |
| 边框 | 冷灰或极浅蓝灰，分隔结构 |
| 成功 | 在线、通过、保存成功 |
| 警告 | 待处理、风险、提醒 |
| 错误 | 失败、危险、删除 |

### 推荐 token

```css
--background: #f8fdff;
--foreground: #102033;
--surface: #ffffff;
--surface-secondary: #f8fafc;
--muted: #5f7184;
--accent: #0ea5e9;
--accent-foreground: #ffffff;
--border: #d7e8f2;
--focus: #38bdf8;
--field-background: #ffffff;
--field-border: #d7e8f2;
--danger: #dc2626;
```

蓝色用于主按钮、当前导航项、键盘焦点描边、关键链接、选中状态和小面积标识强调。

蓝色不用于所有卡片背景、所有边框、所有图标、所有 badge、所有阴影和整页装饰。

一个页面第一眼「全是蓝」时，降低蓝色面积。

## 背景与装饰

默认背景保持干净。

渐变只能非常轻，不能成为页面主角。

不要全站铺明显网格背景。

不要使用装饰性光斑、漂浮圆球、虚化散景或夸张发光。

深色区块只用于公开入口或强强调区域，不用于普通后台内容。

## 排版

界面文案默认使用中文。

英文只用于产品名、技术名词、短标签、代码、接口状态、版本号和行业通用概念。

使用 Geist、Inter 或系统字体。

不要引用受许可限制的商业字体。

数字、ID、运行时版本、时间戳可使用等宽字体。

### 层级

| 场景 | 建议 |
| --- | --- |
| 页面标题 | 24-32px |
| 区块标题 | 18-20px |
| 卡片标题 | 14-16px，500 或 600 |
| 正文 | 14-16px |
| 元信息 | 12-14px |
| 数字、ID、时间戳 | 等宽字体 |

中文和中英混排默认 `letter-spacing: 0`。

不要大面积使用负字距。

不要依赖超粗字体制造层级。

后台页面避免 48px 以上的大标题。

## 布局

同一页面使用一致的 4px 或 8px 间距系统。

常用间距：

- 4px：紧密内部关系。
- 8px：小元素间距。
- 12px：表单 label 与输入框、列表内部。
- 16px：卡片内部基础间距。
- 20px 或 24px：区块间距。
- 32px：页面主要区块间距。
- 48px：大区块或空状态。

后台页面不使用 96px 这类宣传页级大留白。

卡片承载真实信息，不包裹每个 section。

不要卡片套卡片。

卡片圆角建议 8-12px。

弹窗和大面板圆角建议 12-16px。

层级默认依靠边框、背景和间距，少用阴影。

## 组件规则

### Button

- 主按钮使用蓝色。
- 次按钮使用白色或浅灰。
- 危险按钮只用于破坏性操作。
- 图标按钮必须有可访问名称或提示。
- 胶囊按钮用于公开入口主操作、筛选标签、状态标记，不作为后台默认按钮形状。

### Input 和 Form

- 输入框高度建议 40-44px。
- 登录、注册等公开入口可使用更高输入框。
- 输入框必须有 label。
- placeholder 不能替代 label。
- 获得键盘焦点时使用蓝色描边或蓝色边框。
- 错误状态使用红色和可读错误文案。
- 表单字段必须分组。

### Card

- 卡片表达一个明确对象或状态。
- 卡片标题、内容、操作位置保持稳定。
- 没有真实内容时使用空状态。
- 不用假图表、假进度条或假列表制造完成度。

### Chip

- Chip 用于状态、标签、角色、权限。
- Chip 颜色必须有语义。
- 不把 chip 当装饰色块。
- 同一页面避免过多 chip 颜色。

### Table

后台数据优先使用表格或列表。

表格必须具备列标题、合理行高、空状态、加载状态、错误状态和行级操作。

需要查找数据时，提供搜索、筛选或排序。

## 反模式

避免：

- 页面里到处是一次性 Tailwind class。
- 基础控件的尺寸、圆角、焦点状态、悬停状态不统一。
- 所有边框、图标、徽标、背景都使用同一种浅蓝。
- 全页铺明显网格背景。
- 后台页面使用巨幅首屏和宣传文案。
- 卡片里放无意义进度条。
- 放置没有真实含义的预览面板。
- 图标只用于填空。
- 大量空白来自内容缺失，而不是层级设计。
- 中英文随机混排。
- 每个页面重写一套顶部栏、卡片、按钮样式。

## 生成检查清单

每次生成或修改 UI 后检查：

1. 页面主要任务是否一眼可见？
2. 页面是否像后台产品，而不是营销页？
3. 蓝色是否只用于关键强调？
4. 基础控件是否来自统一组件？
5. 重复局部样式是否已提取成组件、variant 或 token？
6. 空状态、加载状态、错误状态是否完整？
7. 是否存在假图表、假进度条或假预览？
8. 中文文案是否自然统一？
9. 桌面和移动布局是否稳定？
10. 页面是否像真实产品，而不是临时 demo？

## 核心原则

参考不是模板。

设计服务任务。

蓝色保持适当。

基础控件保持统一。

业务组件表达结构。

没有真实信息时，宁可诚实空置，也不伪造完成度。
